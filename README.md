# Goroutine Racer

Параллельный "рейсер" URL-адресов, написанный на Go. Утилита принимает список сайтов и возвращает самый быстрый из них по времени ответа, используя конкурентные горутины и механизмы отмены контекста.

## Зачем этот проект

Этот проект создан для отработки ключевых концепций Go, которые отличают уверенного разработчика от новичка:

- Конкурентное выполнение задач через горутины
- Управление жизненным циклом горутин через `context.Context`
- Отмена долгих операций при получении первого успешного результата
- Обработка ошибок в конкурентной среде
- Предотвращение утечек горутин
- Синхронизация доступа к данным
- Работа с флагами командной строки

## Как это работает

Программа запускается со списком URL-адресов и опциональным флагом общего таймаута. Для каждого URL запускается отдельная горутина, выполняющая HTTP-запрос. Как только первый успешный ответ получен, все остальные запросы немедленно отменяются через контекст.

## Установка

```bash
git clone https://github.com/yourusername/goroutine-racer.git
cd goroutine-racer
go build -o racer
```

## Использование

**Базовый запуск:**

```bash
./racer https://google.com https://ya.ru https://bing.com
```

**С указанием общего таймаута:**

```bash
./racer -timeout=3s https://google.com https://ya.ru https://bing.com
```

**Пример с "плохими" URL:**

```bash
./racer -timeout=2s https://google.com https://несуществующий-сайт.рф https://ya.ru
```

## Архитектура

```text
main.go
├── parseFlags() — обработка аргументов командной строки
├── setupContext() — создание контекста с таймаутом
├── raceURLs() — основной оркестратор
│   ├── запуск горутин-воркеров
│   ├── канал результатов
│   └── select с мультиплексированием
└── fetchURL() — функция-воркер для одного URL
```

## Ключевые моменты реализации

**Контекст и отмена**
Родительский контекст с таймаутом передается во все горутины. При получении первого успешного результата вызывается `cancel()` для освобождения ресурсов.

**Каналы для коммуникации**
Используется буферизированный канал для сбора результатов от горутин, чтобы избежать блокировок.

**Проверка `ctx.Done()`**
Каждая горутина перед выполнением запроса и после него проверяет, не был ли отменен контекст.

**Защита от утечек**
Горутины гарантированно завершаются либо после выполнения запроса, либо при отмене контекста. Используется `sync.WaitGroup` для ожидания завершения всех.

**Проверка на data races**
Проект обязательно запускается с флагом `-race` для выявления состояний гонки.

## Проверка работы

Запустите с детектором гонок:

```bash
go run -race main.go -timeout=2s https://google.com https://ya.ru
```

Проверьте отмену запросов (добавьте временной лог в `defer`):

```bash
go run main.go -timeout=1s https://httpbin.org/delay/3 https://ya.ru
```

Убедитесь, что программа не зависает и корректно завершается.
